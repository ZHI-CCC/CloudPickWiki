# 贴图字体

## 💡 功能简介

此功能允许您将文本中的特定字符或词组，替换为 `texture` 文件夹下的自定义图片。它常用于在文本中显示特殊的图标、符号或自定义表情，并且同时支持静态图片（如 `.png`）和动态图片（如 `.gif`）。

> **⭐ 建议：** 虽然贴图字体功能很灵活，但在设计复杂的 UI 界面时，为了更好的性能和控制，我们更推荐您直接使用**图片控件**。

## 📁 配置文件

所有贴图字体的相关设置均在 `config/FontConfig.yml` 文件中进行。

## ⚙️ 配置示例

下面是一个将生僻字 “囖” 替换为在线图片的配置示例。  

```yaml
# 在此文件中，键 (Key) 就是您希望被替换的文字。
囖:
  # texture: 图片的来源路径。
  # 可以是完整的网络 URL，也可以是客户端本地路径 (例如 "texture/icons/coin.png")。
  texture: "http://www.xggames.cn/minecraft/VIP/0.png"
  
  # width: 【必填】贴图在屏幕上显示的实际宽度（单位：像素）。
  width: 10
  # height: 【必填】贴图在屏幕上显示的实际高度（单位：像素）。
  height: 10
  
  # fontWidth: 【必填】该贴图在文本流中占据的横向宽度，用于确保文本对齐。
  # 这决定了图片后面文字的起始位置。
  fontWidth: 9
  
  # offsetX: 贴图的横向偏移量，用于微调位置。正数向右，负数向左。
  offsetX: 5
  # offsetY: 贴图的纵向偏移量，用于微调位置。正数向下，负数向上。
  offsetY: 5
  
  # color: 是否仅替换带有颜色代码的文字。
  # 设置为 true 时，只有像 "§a囖" 这样带颜色的文字才会被替换，普通的 "囖" 字则不受影响。
  color: true
  
  # single: 用于处理多字符词组（例如 "金币"）。
  # false (默认): 将整个词组 "金币" 替换为一张图片。
  # true: 将 "金" 和 "币" 两个字分别进行替换（如果它们各自都有配置）。
  single: false
```

## 📝 参数详解

*   `texture` (字符串):
    定义了图片的来源。可以是 `http://` 或 `https://` 开头的完整网络链接，也可以是客户端 `texture` 文件夹下的相对路径，例如 `textures/gui/my_icon.png`。

*   `width` & `height` (整数):
    **必填项**。这两个参数决定了图片在游戏界面中显示的实际像素尺寸。

*   `fontWidth` (整数):
    **必填项**。这个参数非常关键，它定义了该图片作为一个“字符”在文本中所占据的宽度。合理设置此值可以确保图片前后的文本能够正确对齐，避免重叠或间距过大的问题。

*   `offsetX` & `offsetY` (整数):
    用于对图片的位置进行像素级的微调。`offsetX` 控制水平位置，`offsetY` 控制垂直位置，这对于将图标与文本的基线对齐非常有用。

*   `color` (布尔值):
    一个重要的筛选条件。当设置为 `true` 时，插件只会替换那些明确带有颜色代码的文字（例如 `&a文字` 或 `§c文字`）。这可以有效防止您无意中替换掉所有正常的、无色的目标文字。

*   `single` (布尔值):
    主要用于处理由多个字符组成的词语。以词语“金币”为例：
    *   `single: false`: 插件会将“金币”视为一个整体，查找到它时用一张图进行替换。
    *   `single: true`: 插件会分别检查“金”和“币”这两个字是否有独立的贴图配置，并分别进行替换。通常在处理词语图标时，应使用 `false`。

## 🖼️ 效果展示

配置完成后，当游戏中的文本（例如聊天、物品名称、UI 文本等）中出现符合条件的目标文字时，它就会被自动渲染为指定的图片。
 ![贴图字体](/5.png)
例如，按照上述示例配置，如果您在游戏某处输入了带有绿色代码的文本 `§a囖`，它在屏幕上将不再显示为文字，而是显示为您在 `texture` 链接中指定的那张图片。
