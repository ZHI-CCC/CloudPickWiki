# 远程更新

## 💡 功能简介

该功能允许您通过远程方式为客户端更新资源。它支持使用外部直接下载链接，从而简化了更新流程，无需将资源包体放在更新服务器上。

## 📁 配置文件

您可以在 `config/ResourceConfig.yml` 文件中对远程更新功能进行详细的配置。

```yaml
# 远程更新设置
# 注意：此处的服务端配置仅为示例，实际使用时需要将该文件手动放置到客户端的 CloudPick/config 目录下。

# Mod 版本号
# 用于标识当前 Mod 的版本，以便进行版本对比和更新提示。
modVersion: 1.0.8

# 资源版本号
# 用于标识当前资源文件的版本。
resourceVersion: 1.0.1

# 更新日志对比链接
# 指向一个包含最新版本信息的 yml 文件，用于客户端检查更新。
updateUrl: https://www.yunshimc.com/update.yml

# 资源目录下载链接
# 提供客户端更新所需资源的直接下载地址，通常为一个 .zip 压缩包。
resourceUrl: https://www.yunshimc.com/CloudPick.zip

# 是否解压资源
# 设置为 true 时，客户端在下载完资源压缩包后会自动进行解压操作。
extract: true
```

### 配置项详解

*   `modVersion`: 字符串类型，代表 Mod 本身的版本号。客户端会根据此版本号与 `updateUrl` 中定义的最新版本进行比较，以判断是否需要更新。
*   `resourceVersion`: 字符串类型，代表客户端所使用的资源文件的版本号。
*   `updateUrl`: URL 字符串，指向一个远程的 YAML 文件。该文件应包含最新的 `modVersion` 和 `resourceVersion` 等信息，以便客户端进行比对。
*   `resourceUrl`: URL 字符串，这是一个指向资源压缩包（例如 .zip 文件）的直接下载链接。客户端在检测到需要更新时，会从此链接下载最新的资源。
*   `extract`: 布尔值 (`true` 或 `false`)。当设置为 `true` 时，客户端在成功下载 `resourceUrl` 指定的压缩包后，会自动将其解压到指定目录。如果设置为 `false`，则仅下载而不解压。
